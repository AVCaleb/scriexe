# Exegesis Workspace — Design

**Date:** 2026-07-19
**Status:** Approved pending user review
**Owner:** Caleb

## Purpose

A terminal-based Bible exegesis workspace. One command scaffolds a bilingual
(EN/中文) study file per passage with source texts pre-inserted; the user writes
the exegesis in their editor, with light LLM assistance from **any** coding
agent (Claude Code, Codex, OpenCode, Hermes). All Scripture text comes from a
local, verifiable corpus — never from model memory.

## Requirements

- Local, exact source texts: Greek NT, Hebrew OT, English and Chinese
  translations. All open/free licenses.
- `scaffold <ref>` generates a per-passage study file containing:
  parallel texts, word-study slots (lemma/Strong's/morphology/gloss
  pre-extracted), structure & context sections, interpretation & application
  sections. Headings bilingual EN/中文.
- Agent-agnostic: plain CLI + plain-text corpus + `AGENTS.md` instructions.
  No agent-specific plumbing in the core.
- References accepted in English (`1Pet 3:18-22`, `1 Peter 3:18-22`) and
  Chinese (`彼前3:18-22`) forms.

## Non-goals (YAGNI)

- No commentaries or browsable lexicon modules (Strong's data is used only to
  pre-fill glosses).
- No traditional-script 和合本 initially (simplified only; add later if wanted).
- No TUI/reader app; the terminal + editor + agent is the interface.
- No SWORD/diatheke dependency.

## Architecture

Project root: `/Users/caleb/Projects/exegesis`

```
exegesis/
├── AGENTS.md          # cross-agent instructions; CLAUDE.md is a pointer to it
├── pyproject.toml     # installs one CLI: exeg (Python, minimal deps)
├── src/exeg/
├── data/
│   ├── sources/       # raw downloads (gitignored)
│   ├── corpus/        # normalized plain text, one file per book per version
│   └── corpus.sqlite  # index: verse, lemma, Strong's lookup (rebuildable)
├── studies/           # generated study files, e.g. 1pe_3.18-22.md
└── docs/
```

The normalized corpus is the source of truth and is deliberately greppable
plain text (TSV: `book chapter verse [word-level fields] text`), so any agent
can verify a quotation with ordinary tools. SQLite is a derived index only.

### Datasets (fetched once by `exeg fetch`)

| Corpus | Source | Notes |
|---|---|---|
| Greek NT | MorphGNT / SBLGNT | word-level lemma + full parsing |
| Hebrew OT | OpenScriptures Hebrew Bible (WLC) | word-level lemma + morphology |
| English | WEB, KJV (ebible.org / public domain) | comparison reading |
| Chinese | 和合本 simplified (ebible.org cmn-cu89s) | comparison reading |
| Strong's Greek + Hebrew dictionaries | OpenScriptures | gloss pre-fill only |

Exact download URLs are resolved at implementation time; `fetch` is resumable
and idempotent, and verifies verse counts against canonical tables after
normalization.

### CLI

| Command | Behavior |
|---|---|
| `exeg fetch` | download + normalize all datasets; integrity-check |
| `exeg passage <ref>` | print parallel texts; `--versions` to select |
| `exeg word <G/H-number \| lemma>` | occurrences, gloss, morphology across corpus |
| `exeg search <query>` | search any version; lemma-aware for originals |
| `exeg scaffold <ref>` | generate study file into `studies/` |

## Study file format

Generated by `scaffold` (abridged example for 1 Pet 3:18–22):

```markdown
# 1 Peter 3:18–22 · 彼得前书 3:18–22
> scaffolded 2026-07-19 · SBLGNT | WEB | KJV | 和合本

## Text · 经文对照
### v. 18
- **SBLGNT** ὅτι καὶ Χριστὸς ἅπαξ περὶ ἁμαρτιῶν ἔπαθεν…
- **WEB** Because Christ also suffered for sins once…
- **KJV** For Christ also hath once suffered for sins…
- **和合本** 因基督也曾一次为罪受苦…

## Word Studies · 字词研究
### πάσχω (ἔπαθεν, v. 18) — G3958 · aorist act. ind. 3sg
gloss: to suffer · in 1 Peter: 11× (2:19, 2:20, 2:21, …)
（your analysis · 你的分析）

## Structure & Context · 结构与背景
- Literary structure · 文学结构：
- Historical setting · 历史背景：
- Place in the book's argument · 在全书论证中的位置：

## Interpretation · 释经结论

## Theology & Application · 神学综合与应用
- Cross-references · 串珠：
```

Word-study entries are auto-selected: verbs plus theologically weighted nouns
(selection heuristic tuned during implementation), deduplicated per passage,
each pre-filled with lemma, Strong's number, parsing, gloss, and book-level
occurrence references. Analysis prose is left blank for the user/agent.

## AGENTS.md (cross-agent contract)

- Quote Scripture only via `exeg` or the `data/corpus/` files; never from
  model memory. Cite a verse reference for every textual claim.
- Agents may draft in Word Studies and Structure & Context sections when
  asked; the Interpretation section is the user's unless explicitly requested.
- Bilingual convention: mirror the file's EN/中文 heading style in drafted
  content.
- `CLAUDE.md` contains only a pointer to `AGENTS.md` so Claude Code, Codex,
  OpenCode, and Hermes all follow the same rules.

## Error handling

- Unknown reference → error listing closest book-name matches (EN and 中文).
- Corpus not fetched → clear message to run `exeg fetch`.
- Verse absent from a version (textual variants) → explicit `[not in SBLGNT]`
  marker in output; never silently skipped.
- `scaffold` refuses to overwrite an existing study file without `--force`.

## Testing

- pytest.
- Golden tests for reference parsing (English and Chinese forms, ranges,
  cross-chapter ranges).
- Corpus integrity: verse counts per book checked against canonical tables
  after `fetch`/normalization.
- Golden-file test for `scaffold` of a fixed passage.
- Unit tests for `word`/`search` on small fixture corpora.

## Decisions log

- Custom CLI over SWORD/diatheke: greppable plain-text corpus is the strongest
  form of agent-agnosticism; exact control of bilingual markdown output;
  avoids OSIS/ThML markup cleaning. (User approved 2026-07-19.)
- Template sections: all four (parallel texts, word studies, structure &
  context, interpretation & application). Headings bilingual EN/中文.
- LLM's role is light assistance on top of the scaffold, not generation of
  the exegesis.
